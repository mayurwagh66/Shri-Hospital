# DEPLOYMENT QUICK REFERENCE CARD

## üìã MONGODB ATLAS SETUP

```
1. Go to: https://www.mongodb.com/cloud/atlas
2. Create Account ‚Üí Create Cluster (M0 Free)
3. Database Access ‚Üí Add User:
   Username: hospital_admin
   Password: [Strong password]
4. Network Access ‚Üí Add IP: 0.0.0.0/0
5. Clusters ‚Üí Connect ‚Üí Connection String:
   mongodb+srv://hospital_admin:PASSWORD@cluster.mongodb.net/shri-hospital
6. SAVE THIS STRING!
```

## üîê GENERATE SECRETS

```bash
# Run these commands in terminal:

# JWT Secret
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Session Secret
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Save both outputs!
```

## üöÄ DEPLOY TO VERCEL

```
1. Go to: https://vercel.com/dashboard
2. Click: "Add New" ‚Üí "Project"
3. Select: "shri-hospital" repo ‚Üí Import
4. Wait for deployment
5. Add Environment Variables:
   
   MONGODB_URI = [from MongoDB Atlas]
   JWT_SECRET = [32-char secret]
   SESSION_SECRET = [32-char secret]
   NODE_ENV = production
   
6. Redeploy ‚Üí Wait for "Ready" ‚úì
```

## üêõ TROUBLESHOOTING

| Error | Check |
|-------|-------|
| MongoDB Connection Failed | IP whitelist, credentials, connection string |
| Cannot Find Module | package.json, npm install locally |
| 404 on Pages | views/ folder, file names |
| Timeout 504 | Database query optimization |
| Env Vars Not Working | Vercel Settings ‚Üí Environment Variables |
